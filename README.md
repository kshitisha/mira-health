# MIRA — Medical Intelligence Robotic Automation

An AI-powered health prediction platform that analyses patient blood test results and generates intelligent health risk assessments using **Groq (LLaMA 3)**.

---

## Tech Stack

| Layer     | Technology                                  |
|-----------|----------------------------------------------|
| Frontend  | React 19 · Vite · Tailwind CSS              |
| Backend   | FastAPI · Python 3.10 · SQLAlchemy          |
| Database  | SQLite (persistent)                          |
| AI/ML     | Groq API — LLaMA 3 (llama3-8b-8192)        |
| Deploy    | Vercel (frontend) · HuggingFace Spaces (backend) |

---

## Features

- **Full CRUD** — Create, Read, Update, Delete patient records
- **AI Health Assessment** — Groq LLaMA 3 analyses Glucose, Haemoglobin, and Cholesterol values to generate a clinical risk assessment in the Remarks field
- **Auto-regeneration** — Updating blood values triggers a fresh AI assessment automatically
- **Input Validation** — Email format, future DOB prevention, numeric range checks on all blood values
- **Persistent Storage** — SQLite via SQLAlchemy ORM
- **Clinical Indicators** — Visual Normal / Borderline / High Risk badges per blood marker
- **Dashboard** — At-a-glance stats across all patients

## Local Development

### Backend

```bash
cd backend
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

# Create .env from template
cp .env.example .env
# Add your GROQ_API_KEY in .env

uvicorn main:app --reload --port 7860
```

API docs available at: http://localhost:7860/docs

### Frontend

```bash
cd frontend
npm install

# Create .env from template
cp .env.example .env
# Set VITE_API_BASE_URL=http://localhost:7860

npm run dev
```

App runs at: http://localhost:5173

## Deployment

### Backend → HuggingFace Spaces

1. Create a new Space (Docker SDK)
2. Push the `backend/` folder contents
3. Add `GROQ_API_KEY` as a Space Secret (Settings → Variables and secrets)

### Frontend → Vercel

1. Push `frontend/` to GitHub
2. Import repo in Vercel
3. Set env var: `VITE_API_BASE_URL=https://your-hf-space.hf.space`
4. Deploy

## API Endpoints

| Method | Endpoint                              | Description                        |
|--------|---------------------------------------|------------------------------------|
| GET    | `/patients`                           | List all patients                  |
| POST   | `/patients`                           | Create patient + AI assessment     |
| GET    | `/patients/{id}`                      | Get single patient                 |
| PUT    | `/patients/{id}`                      | Update patient (re-runs AI if blood values change) |
| DELETE | `/patients/{id}`                      | Delete patient                     |
| POST   | `/patients/{id}/regenerate-remarks`   | Manually trigger AI re-assessment  |

---

## Patient Data Fields

| Field         | Type    | Validation                          |
|---------------|---------|--------------------------------------|
| Full Name     | string  | Min 2 chars                         |
| Date of Birth | date    | Cannot be future date               |
| Email         | string  | Valid email format, unique          |
| Glucose       | float   | Positive, max 600 mg/dL             |
| Haemoglobin   | float   | Positive, max 25 g/dL              |
| Cholesterol   | float   | Positive, max 700 mg/dL             |
| Remarks       | string  | Auto-generated by MIRA AI           |
